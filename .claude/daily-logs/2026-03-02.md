### Sesión 2 — Bug fixes (POST 400 + GET 500)

**Bug 1: POST /api/agenda/tasks → 400 "Error al crear tarea"**
- Causa: `dueDate` del modal se enviaba como `'yyyy-MM-dd'` o `'yyyy-MM-ddTHH:mm:ss'` (sin timezone), pero `z.string().datetime()` requiere ISO completo con timezone
- Fix en `lib/validations/agenda-tasks.ts`: `z.preprocess()` normaliza ambos formatos → agrega `T00:00:00.000Z` o `.000Z` según el caso
- Fix también aplicado en `app/api/agenda/tasks/[id]/route.ts` (schema de PUT)

**Bug 2: GET /api/agenda/tasks → 500**
- Causa probable: `_count.select.subtasks` en el `findMany` podía fallar en contexto Next.js aunque funcionaba en Node.js standalone
- Fix en `app/api/agenda/tasks/route.ts`:
  - Eliminado `subtasks: true` del `_count.select` del `findMany`
  - Reemplazado por dos queries `groupBy` en paralelo (total + done) dentro de try-catch
  - Si las queries fallan → `subtasks: 0, subtasksDone: 0` como fallback silencioso

### Sesión 15 — Fix T2 Database: Prisma client-t2 no generado

**Problema**: `POST /api/compras/comprobantes` retornaba 503 "Base de datos T2 no disponible" cuando el usuario estaba en modo Extended (Ctrl+Z activado).

**Diagnóstico**:
1. `DATABASE_URL_T2` en `.env` ✓
2. `companyViewConfig` para company 3 con `enabled: true` y `t2DbEnabled: true` ✓
3. `require('@prisma/client-t2')` → `Cannot find module '@prisma/client-t2'` ✗

**Root cause**: `@prisma/client-t2` nunca fue generado. En `lib/prisma-t2.ts`, el `require('@prisma/client-t2')` falla silenciosamente (try/catch) → `PrismaClientT2 = null` → `isT2DatabaseConfigured()` retorna `false`.

**Fix**: `npx prisma generate --schema=prisma/schema-t2.prisma` → cliente generado en `node_modules/@prisma/client-t2` (57ms). Requiere restart del dev server.
