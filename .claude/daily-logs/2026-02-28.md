### Mantenimiento / Infraestructura
- **Fix crítico: Prisma engine gate compartido via globalThis** — El `engineGate` se almacenaba como variable de módulo, causando que cada route de webpack creara su propio gate (N módulos × 40 retries = 400+ `$queryRaw` concurrentes que saturaban el engine). ## Log Técnico — Sesión 1

### Prisma Engine Gate → globalThis (lib/prisma.ts)
**Problema**: En dev, Next.js compila cada route como módulo webpack separado. La variable `let engineGate = startEngineGate()` se duplicaba por cada módulo, generando cientos de `$queryRaw\`SELECT 1\`` concurrentes que saturaban el engine antes de que pudiera arrancar. Esto causaba cascade: `/api/auth/me` → 401 (Prisma error tratado como auth error) → refresh 500 → kick a login.

**Cambios**:
1. `globalForPrisma` ahora incluye `engineGate: Promise<void> | undefined`
2. `startEngineGate()` solo se ejecuta si `!globalForPrisma.engineGate`
3. Reducido de 40 retries × 250ms a 20 retries × 500ms (mismos 10s, menos spam)
4. Reconexión mid-session también actualiza `globalForPrisma.engineGate`

**Archivos**: `lib/prisma.ts`

## Log Técnico — Sesión 2

### Fix /api/maintenance/costs 500 persistente — Migración faltante

**Problema**: El endpoint retornaba 500 en todo momento (no era cold start — persistía incluso en `npm start`). Auth pasaba correctamente. El error venía de Prisma queries.

**Causa raíz**: Las tablas `maintenance_cost_breakdowns`, `technician_cost_rates`, `third_party_costs` y `maintenance_budgets` **nunca tuvieron una migración Prisma**. Solo existían en `full_schema.sql` (usado para instancias desde cero). La DB real no tenía esas tablas → cualquier query lanzaba un error de "relation does not exist".

**Fix**: Creada migración `prisma/migrations/20260228_add_maintenance_cost_system/migration.sql` con:
- `CREATE TABLE IF NOT EXISTS` para las 4 tablas
- Indexes idempotentes (`CREATE ... IF NOT EXISTS`)
- FKs con `DO $$ BEGIN...EXCEPTION WHEN duplicate_object THEN null; END $$`
- Aplicada con `npx prisma migrate deploy` — confirmado: "All migrations have been successfully applied."

**Archivos**: `prisma/migrations/20260228_add_maintenance_cost_system/migration.sql`

**Nota**: El `build` script ya corre `prisma migrate deploy` automáticamente (`"build": "prisma migrate deploy && prisma generate && next build"`), así que de ahora en adelante se aplica solo al construir.
