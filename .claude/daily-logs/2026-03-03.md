### Decisions Made

- **Dual auth** (`lib/chat/auth.ts`): todos los endpoints de chat soportan cookie (web) Y Bearer token (mobile) en un solo helper
- **No withGuards**: endpoints de chat usan auth propio para soportar Bearer. Se podría unificar con withGuards en el futuro
- **Fire-and-forget para Pusher/Push**: las notificaciones no bloquean el response del POST message
- **Transaction atómica**: create message + update conversation denormalized + increment unread en una sola transaction
- **Expo Router**: src/app/ como root para file-based routing
